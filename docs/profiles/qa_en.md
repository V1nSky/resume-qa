## Positioning
Junior QA / QA-minded developer who can test not only UI, but also business logic, APIs, statuses, queues, forms, Telegram flows, and edge cases.

## Short website summary
I test products both as a user and as a developer: I verify UI/UX flows, write Jira-style bug reports, prepare checklists and test cases, test APIs, statuses, queues, forms, authentication, and negative scenarios. I can review my own backend/Telegram projects and identify release risks before shipping.

## Core QA skills
- Bug reports: steps, actual result, expected result, severity, priority, environment.
- Functional testing: forms, CRUD, admin panels, roles, statuses, queues, payments, booking flows.
- API testing: REST endpoints, validation, auth, error handling, rate limiting.
- Regression testing: fixes and critical user paths.
- Business logic testing: publishing, booking, payments, deduplication, scheduling, exports.
- Negative testing: invalid data, expired sessions, empty states, repeated actions, unavailable services.
- Test documentation: checklists, test cases, bug reports, traceability matrix.
