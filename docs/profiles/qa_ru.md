## Сильные QA-навыки
- Bug reports: steps, actual result, expected result, severity, priority, environment.
- Functional testing: формы, CRUD, админки, роли, статусы, очереди, платежные и booking-flow сценарии.
- API testing: REST endpoints, validation, auth, error handling, rate limiting.
- Regression testing: проверка фиксов и критических пользовательских маршрутов.
- Business logic testing: публикации, бронирования, оплата, дедупликация, расписание, экспорт.
- Negative testing: неверные данные, истекшие сессии, пустые состояния, повторные действия, недоступные сервисы.
- Test documentation: checklists, test cases, bug reports, traceability matrix.
